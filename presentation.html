Core Innovation #1

3-Phase Extraction Pipeline

Inspired by TrustGraph's ontology-guided approach, powered by neo4j-graphrag + Gemini.

Phase A — Discovery

Schema-Free Extraction

Gemini freely discovers all entities and relationships with no constraints. Maximum coverage — nothing is missed.

Output: Raw graph + discovered schema

Phase B — Ontology

OWL Generation

LLM generates a formal ontology in Turtle format from discovered patterns. Consolidates types, defines hierarchy, sets constraints.

Output: Validated OWL ontology

Phase C — Precision

Ontology-Guided Re-extraction

Re-extracts constrained to the ontology. Per-chunk ontology subsetting keeps prompts small. Entity resolution deduplicates.

Output: Clean, structured KG

Core Innovation #2

Agentic GraphRAG

Not just semantic search — 8 specialized query tools the agent dynamically selects based on question type.

Concept Search Vector similarity on entity embeddings
Entity Explorer 1-2 hop neighborhood traversal
Path Finder Shortest path between any two entities
Text2Cypher LLM-generated Cypher for complex queries
Hybrid Search Vector + graph traversal combined
Communities Cluster detection + LLM summaries
Ontology Info Schema introspection for meta-questions
Graph Stats Distribution, connectivity, growth metrics

Pre-built Cypher templates for common patterns (fast + reliable). Text2Cypher only for complex analytical questions. Best of both worlds.

Core Innovation #3

"Thinking Through the Graph"

Watch the AI reason in real time. Three animation modes combine into an unprecedented UX.

Sequential Path Glow

As the agent traverses nodes during a query, each node lights up in sequence — showing the exact reasoning path through the graph.

Ripple Expansion

When exploring neighborhoods, concentric ripples expand outward from the query origin, revealing connections in layers.

Thought Stream

A floating overlay displays the agent's internal reasoning steps as they happen — "Searching for Einstein... Found 12 connections..."

Backend emits thinking_* WebSocket events during tool execution — the graph animates before the answer arrives.

Core Innovation #4

Voice-Controlled Ontology Editing

Refine the knowledge structure by speaking naturally. The system re-extracts automatically.

Say it → it happens

"Add a new entity type called Project"
  → add_class("Project", "Thing")
"Connect Person to Organization through 'works at'"
  → add_object_property("worksAt", "Person", "Org")
"Now re-scan documents for the new Project type"
  → trigger_re_extraction("new_types")

OntologyAgent: 13 tools

  • Add, remove, rename entity classes
  • Define object & datatype properties
  • Set class hierarchy (subClassOf)
  • Validate consistency with rdflib
  • Selective re-extraction after changes
  • OWL serialized as Turtle, manipulated as JSON

User Experience

A conversation, not a query

// User speaks:
"How does Einstein relate to quantum mechanics?"

// Gemini Live understands intent → selects find_path tool
find_path("Einstein", "Quantum Mechanics", max_hops=5)

// Backend executes Cypher → finds path
// Emits thinking_traverse events → graph animates
highlight_nodes(["einstein", "photoelectric", "quantum"])

// Gemini narrates the result:
"Einstein connects to Quantum Mechanics through the
 Photoelectric Effect. His 1905 paper proposed light
 quanta, which became foundational to quantum theory.
 Would you like me to expand any of these nodes?"

  • Follow-up conversation continues naturally
